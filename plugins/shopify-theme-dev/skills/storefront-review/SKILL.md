# Storefront Review

For audits, return findings first, ordered by severity, with exact file or URL evidence. Separate source-confirmed issues from rendered-page inferences.

Inspect only the relevant page, template, section, blocks, snippets, assets, and current contracts. Review hierarchy, intent, trust, copy, CTAs, headings, landmarks, metadata, canonical behavior, JSON-LD, visible FAQ parity, internal links, keyboard/focus, contrast, touch targets, reduced motion, mobile overflow, LCP, CLS, INP, media dimensions/loading, and theme-editor compatibility.

Use the in-app Browser for routine local responsive inspection. Reserve Chrome DevTools MCP for console, network, tracing, performance, or deep debugging.

When review follows implementation, report validation evidence and remaining production risk. Do not duplicate the full brand system inside the review.
